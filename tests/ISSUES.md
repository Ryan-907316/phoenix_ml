# Issue register

The single living list of known problems: confirmed bugs, suspected risks, test-coverage gaps, and pre-release QA items.

**How to use this file**
- Work top-down: priority reflects blast radius (does it corrupt the artifact a user deploys? does it fail silently?) more than likelihood.
- When an item is resolved: move it to the Resolved section, ~~strike it through~~, add the date and the regression test that now guards it. Never just delete, as the resolved list is the project's defect history (and in future releases, will be used for CHANGELOG.md)
- When a new problem is found: add it here first, tagged and prioritised, even if it's fixed in the same sitting.

**Tags**
- `[BUG]`: confirmed defect, traced to specific code; fixing it changes behaviour.
- `[RISK]`: plausible failure mode, but not yet reproduced/confirmed.
- `[TEST-GAP]`: behaviour believed correct but unprotected by any test.
- `[QA]`: behaviour unconfirmed, needs more thorough and manual review.
